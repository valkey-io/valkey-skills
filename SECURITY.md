Security Policy
===============

## Reporting a Vulnerability

If you believe you have found a security vulnerability in valkey-skills, or in
the Valkey behavior described by these skills, contact the Valkey team at
security@lists.valkey.io.

Do not create a public GitHub issue, pull request, or discussion for security
reports.

This repository contains AI skill documentation rather than runtime code.
Public issues are appropriate for non-security content bugs, stale references,
or misleading examples. If a report could help someone exploit, bypass, or
weaken a Valkey deployment, use the private security contact instead.

The Valkey project follows responsible disclosure. Depending on severity, the
team may coordinate with Valkey vendors before public disclosure. To be added
to the vendor notification list, contact maintainers@lists.valkey.io.
